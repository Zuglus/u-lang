## ФАКТ 1: Дженериков НЕТ в грамматике

**Где:** `transpiler/src/u.pest`

**Факт:** `identifier = @{ !keyword ~ (LETTER | "_") ~ (LETTER | ASCII_DIGIT | "_")* }`

`[` и `]` — не входят в identifier. `Option[T]` не может распарситься.

**Дата:** u.pest — Apr 2, пример option_some_none.u — Apr 4.

**Вывод:** Примеры с дженериками описывают ЖЕЛАЕМОЕ, не реальное.

## ФАКТ 2: None — отдельный тип, не вариант enum

**Где:** `transpiler/src/type_checker.rs:6-17`

```rust
pub enum Type {
    Int,
    Float,
    String,
    Bool,
    None,  // <-- отдельный тип!
    List(Box<Type>),
    Struct(String),
    Enum(String),
    ...
}
```

**Вывод:** `none` — это как `null` в других языках. Не вариант `Option[T]`.

**Противоречие со спецификацией:** В SPEC `None` — вариант `Option[T]`.

## ФАКТ 3: Генерация enum работает без дженериков

**Где:** `transpiler/src/generator.rs:594-604`

Генерируется: `enum Name { Variant(Type, Type), }`

Нет `<T>` — только конкретные типы в полях.

**Пример:**
```u
enum Option
    Some(value: Int)    # конкретный Int!
    None()              # пустой кортеж
end
```

Генерируется в Rust:
```rust
enum Option {
    Some(Int),
    None(()),
}
```

**Где:** `transpiler/src/generator.rs:760-770`

```rust
if name == "None" {
```

## ВЫВОД

**Документация опережает реальность на 2+ дня.**

SPEC.md и примеры описывают ЦЕЛЕВОЕ состояние, не текущее.

**Критический разрыв:**
1. Нет дженериков → `Maybe[T]` невозможен
2. `None` — не вариант, а тип → философия Being/Nothing не реализована
3. Нет `Box[T]` → нельзя сделать AST для self-hosting

**Что нужно для целостности:**
1. Добавить `[T]` в грамматику
2. Переделать `None` из типа в вариант
3. Добавить `Box[T]` для рекурсии
4. Или признать: текущая реализация — прототип, не соответствует философии
