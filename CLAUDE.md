# CLAUDE.md — контекст для Claude Code

## Проект

U Language — язык над Rust для ИИ-генерации. Код отражает реальность: `thing` (нечто с жизнью) и `process` (действие во времени). `.u` и `.rs` в одном проекте.

## Спецификация

`spec/SPEC.md` (v2.1+). Примеры в `examples/`.

## Архитектура

```
.u файл → [Parser/pest] → AST → [Generator] → .rs → rustc → бинарник
```

- `u run` — компиляция + запуск + кэш
- `u build` — компиляция через Rust, нативная скорость
- `u test` — запуск `test process`
- `u check` — только парсинг (AST)
- `u fmt` — форматирование

## Диалектика языка — Бытие, Ничто, Становление

Реальность существует во времени двумя способами:

- **Нечто** (`thing`) — сохраняется во времени. Три момента жизни: `being` (приход), `processes` (жизнь через процессы), `nothing` (уход).
- **Процесс** (`process`) — разворачивается во времени. Берёт входные нечто, разворачивает тело, даёт выходное нечто.
- **Becoming[T]** с вариантами `Being(value)` / `Nothing` — становление нечто типа T. Гегелевская триада бытие-ничто-становление буквально в системе типов.

Скрипт — неявное нечто: процесс ОС с неявным `processes` без `being` и `nothing`.

### Формула выхода процесса

Процесс всегда оставляет след — без следа его бы не было. Три формы выхода через единый словарь Being/Nothing:

| Куда направлено изменение | Сигнатура |
|---|---|
| Новое нечто программы | `-> Becoming[T]` — становление, может стать |
| Во входное нечто | `mut` в параметре, `-> Nothing[T]` |
| Во внешнее нечто (stdout, fs, net) | `-> Nothing[T]` без mut |

`Nothing[T]` всегда означает: нового нечто типа T не возникло. T — тип того, что было бы результатом если бы рождалось.

### Ядро в .u файлах

`print`, `read_file`, `sleep` — обычные `process`-определения в `core/*.u`, тело которых вызывает Rust-обёртки из `core/*.rs`. Никакого хардкода в генераторе. Правило: процесс определяется либо в `.u`, либо в `.rs` через `use`. Третьего пути нет.

Строковая интерполяция `$name` — свойство литерала, не процесса. Работает везде одинаково.

## Ключевые решения

- **`process`** вместо `fn`, **`thing`** вместо `struct`
- **`being` / `processes` / `nothing`** — жизненный цикл нечто
- **`Becoming[T]`** — становление нечто (вместо Maybe/Option)
- **`mut` для мутации**: в определении процесса/метода, при вызове — всегда `.`
- **`end`-блоки**: вместо `{}`
- **Модификаторы**: `pub`, `test`, `mut`, `unsafe`, `weak`
- **Лямбда**: `x => expr`, `(x, y) => expr`
- **`!`** force-unwrap (panic). `?` убран.
- **`$name`** / **`$(expr)`** строковая интерполяция
- **Return**: явный. Нет return = процедура
- **Типы**: `pub process` — с типами и `->`, остальные — без
- **Spawn безопасен**: паники ловятся, `mut` на внешних данных запрещён
- **Именование**: заглавная = тип (`Sqlite`, `Channel`), строчная = процесс (`print`, `read_file`)
- **30 ключевых слов**: process, thing, being, processes, nothing, return, if, elif, else, end, for, in, while, loop, break, continue, impl, trait, enum, match, pub, test, unsafe, weak, use, spawn, and, or, not, mut

## Четыре уровня доступности

1. **Язык** — `Int`, `String`, `Bool`, `List`, `Becoming`, `Result`, `true`, `false`
2. **Ядро** (без use) — `print`, `read_file`, `write_file`, `list_dir`, `create_dir`, `sleep`, `range`, `len`, `str`, `int`
3. **Стандартная** (`use std.xxx`) — `Sqlite`, `Router`, `Channel`, `Args`, `Json`, `Regex`
4. **Внешнее** (`use`) — `.rs` модули, crates.io

## Команды

```
cd transpiler
cargo build                     # собрать
cargo test                      # тесты транспилятора
cargo install --path .          # установить как `u`
u run examples/hello.u          # компиляция + кэш
u build examples/hello.u        # компиляция → нативный бинарник
u check examples/hello.u        # только парсинг (AST)

u fmt examples/hello.u          # форматирование
```

## Приоритет работы

1. Разрешение противоречий между спекой и реализацией (`fn`/`struct` → `process`/`thing` в парсере и генераторе)
2. Генератор: поддержка `thing` с `being/lives/nothing`
3. Парсер: новые ключевые слова
4. Новые примеры из реальных задач
5. Маппинг ошибок: rustc → .u файл

## Структура транспилятора

```
transpiler/src/
  main.rs          — CLI (clap): u run / u build / u check / u test / u fmt
  lib.rs           — pub mod ast, parser, generator, formatter
  ast.rs           — AST типы
  parser.rs        — pest парсер → AST
  u.pest           — PEG грамматика
  generator.rs     — AST → Rust-код
  formatter.rs     — u fmt форматирование
```

## Метод работы

Искать противоречия между тем что заявлено и тем что есть. Разрешать их через возврат к конкретному: что именно в коде, в спеке, в примере. Не плодить абстракций. Код — отражение реальных связей, не изобретённая архитектура.
