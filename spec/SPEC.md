# U — язык программирования

## Скорость Rust. Простота для ИИ.

Версия 2.1 — март 2026

---

## 1. Что это

U — транспилятор в Rust. Простой код на U превращается в оптимизированный Rust автоматически. ИИ генерирует U-код, транспилятор генерирует Rust, компилятор Rust создаёт нативный бинарник.

Не интерпретатор. Не виртуальная машина. Компиляция в нативный код через Rust.

### 1.1. Доказано

| Факт | Числа |
|------|-------|
| Скорость vs Rust | 93% (0.44 vs 0.41 сек на 1000 файлов) |
| Скорость vs Python | 4.5x быстрее |
| HTTP-сервер (через .rs модуль) | 50,243 req/sec |
| vs Axum (Rust-фреймворк) | 2.3x быстрее |
| Строк кода vs Rust | 22% меньше (137 vs 176) |
| Повторный запуск | 0.01 сек (кэш) |
| Первый запуск (hello) | ~0.5 сек (без тяжёлых зависимостей) |
| Один бинарник | Да |

### 1.2. Для кого

ИИ генерирует код. Человек описывает задачу. Язык проектируется для:

- Минимум токенов → меньше ошибок ИИ
- Однозначный синтаксис → предсказуемая генерация
- Строгие правила → компилятор ловит ошибки → быстрая обратная связь ИИ
- Нативная скорость → результат сразу быстрый

### 1.3. Архитектура: простое → сложное

Язык построен по принципу композиции: сложное складывается из простых элементов. Каждый уровень строго определён и использует только предыдущие.

**Уровень 0: Атомы (скаляры)**
- `Int` — целые числа
- `Float` — числа с плавающей точкой
- `String` — строки UTF-8
- `Bool` — `true`, `false`
- `none` — отсутствие значения

**Уровень 1: Составные типы**
- `List[T]` — динамический массив (все элементы одного типа)
- `struct` — записи с именованными полями
- `enum` — перечисления с вариантами (`Some(T) | None`)

**Уровень 2: Функции**
- `fn` — именованные функции
- `fn(x) expr` — замыкания (лямбды)
- Вызов: `name(args)` или `obj.method()`

**Уровень 3: Управление потоком**
- `if / elif / else` — условное выполнение
- `match` — сопоставление с образцом (включая списки: `[x, ..rest]` — x первый, rest остальные)
- `for`, `while`, `loop` — циклы
- `range(start, end)` — итераторы диапазонов

**Уровень 4: Модули**
- Файл = модуль
- `use path: name` — импорт
- Всё публично внутри файла (MVP)

**Уровень 5: Конкурентность**
- `spawn` — запуск задач
- `Channel` — каналы для коммуникации
- `recv_timeout(ms)` — предсказуемая синхронизация

**Принцип композиции:**
- `Option[T]` = `enum { Some(T), None }` (уровень 1)
- `Result[T, E]` = `enum { Ok(T), Err(E) }` (уровень 1)
- Кортежи не нужны — используйте `struct` (явнее для ИИ)
- `Map[K,V]` не включён — используйте `List[(K,V)]` или `.rs` модули

### 1.4. Принципы проектирования

- Сложность от задачи, не от языка
- Каждая фича убирает конкретную проблему
- Проверяется бенчмарками, не теориями
- Локальность ошибок: ошибка компиляции указывает на место вызова/использования, не на определение

### 1.5. Семантика для LLM

Язык проектируется так, чтобы LLM мог «в уме» проследить выполнение кода без запуска. Это снижает количество итераций генерации с 3-5 (Python/Rust) до 1.

**Правила предсказуемости:**

| Правило | Почему важно |
|---------|-------------|
| **Нет перегрузки операторов** | `+` всегда числа, `++` всегда строки — не надо анализировать типы |
| **Нет implicit conversions** | `Int + Float` — ошибка, нужно явное `to_float()` — нет магии |
| **Строгий порядок evaluation** | `foo() + bar()` — всегда слева направо — побочные эффекты предсказуемы |
| **Нет nil/null** | Всегда `Option[T]` — `Some(x)` или `None` — проверка обязательна |
| **Нет exceptions** | Всегда `Result[T, E]` — ошибки это значения, не прерывания потока |
| **Всё выражение имеет значение** | `if` возвращает значение, нет различия statement/expression |
| **Отсутствие глобального состояния** | Все зависимости явно передаются — поведение локально |
| **Compile-time циклы** | Циклические ссылки отклоняются компилятором — нет runtime сюрпризов |
| **Deterministic drop** | Память освобождается в обратном порядке создания — предсказуемо |

**Результат:** LLM генерирует корректный код с первой попытки, потому что семантика однозначна и проверяется статически.

### 1.5. Диалектика дизайна

Каждое противоречие разрешается через инструмент прозрачности:

| Противоречие | Тезис | Антитезис | Синтез (инструмент) |
|--------------|-------|-----------|---------------------|
| **Скорость HTTP** | U-lang быстрый | Реально это Rust | Честная интеграция: `hyper` + `tokio`, бенчмарки публичны (50k req/sec) |
| **Параллельность** | Автоматика удобна | Непредсказуемость опасна | `par_for` — явный маркер. Без него — последовательно, с ним — параллельно |
| **Запреты без причин** | Список простоты | Каждый запрет нуждается в обосновании | Каждый пункт — конкретная проблема. Пример: «нет макросов → макросы скрывают код → ИИ ошибается → отладка сложнее» |
| **Скрытая магия** | Простота (не писать `&mut`) | Нужно понимать что происходит | `u show-generated` — показывает сгенерированный Rust. Простота в коде, явность по запросу |
| **Память** | Автоматика управления | Утечки возможны | `u profile` — встроенный профилировщик: allocations, cycles, lifetime |

**Вывод:** Противоречия разрешаются не запретами, а инструментами контроля.

## 2. Инструменты

```

u run script.u        # повторно — из кэша (0.01 сек)
u build script.u      # бинарник для деплоя
u test script.u       # запуск test fn
u test                # все тесты в проекте
u check script.u      # парсинг, проверка ошибок

`u run` = `u build` + execute + cache. Один компилятор, одна среда.

### 2.1. Умные зависимости

Транспилятор анализирует AST и подключает только нужное:

| Скрипт использует | Зависимости | Время первой сборки |
|-------------------|------------|-------------------|
| print, if, for, файлы | Ничего | ~0.5 сек |
| parse_json | serde_json | ~0.8 сек |
| Sqlite | rusqlite | ~1.2 сек |
| spawn, сервер | tokio, hyper | ~1.5 сек |

Повторная сборка (код изменён): ~0.5 сек. Без изменений: 0.01 сек (кэш).

## 4. Доступность

### 4.1. Типы данных

Язык имеет строгую иерархию: сложные типы строятся из простых.

#### 4.1.1. Скаляры (уровень 0)

| Тип | Описание | Пример |
|-----|----------|--------|
| `Int` | 64-битное целое | `42`, `-5` |
| `Float` | 64-битное с плавающей точкой | `3.14`, `-0.5` |
| `String` | UTF-8 строка | `"hello"` |
| `Bool` | Булево | `true`, `false` |
| `none` | Отсутствие значения | `list.first()` может вернуть `none` |

#### 4.1.2. Составные типы (уровень 1)

**List[T]** — динамический массив однотипных элементов:
```u
numbers = [1, 2, 3]        // List[Int] — тип выведен из элементов
names = ["a", "b"]         // List[String] — тип выведен из элементов
empty: List[Int] = []      // Пустой список — требуется явный тип
```

**Вывод типов:**
- Тип выводится автоматически из инициализатора
- Пустые коллекции (`[]`) требуют явной аннотации типа
- Это предотвращает ошибки AI при неоднозначных ситуациях

**struct** — запись с именованными полями:
```u
struct Point
    x: Int
    y: Int
end

p = Point(x: 10, y: 20)
print(p.x)                 // 10
```

**enum** — алгебраический тип с вариантами:
```u
enum Option[T]
    Some(T)
    None
end

enum Result[T, E]
    Ok(T)
    Err(E)
end
```

#### 4.1.3. Кортежи (ограниченные)

Кортежи поддерживаются **только** в `for` для распаковки пар:
```u
pairs = [(1, "a"), (2, "b")]    // List структур или List[(Int, String)]
for (num, char) in pairs
    print("$(num): $(char)")
end
```

Явный тип `Tuple[A, B]` не существует — используйте `struct` (явнее для ИИ).

#### 4.1.4. Отсутствие Map

Тип `Map[K, V]` **не включён** в язык. Альтернативы:

1. **Малые данные** — `List[(K, V)]` + поиск:
```u
config = [("host", "localhost"), ("port", "8080")]
host = config.filter(fn(pair) pair[0] == "host").first()
```

2. **Конфигурация** — `struct` (предпочтительно):
```u
struct Config
    host: String
    port: Int
end
```

3. **Большие данные** — `.rs` модуль с `HashMap`.

#### 4.1.5. Типы по умолчанию

```u
Int, Float, String, Bool
List[T]
Result[T, E]
none, true, false
```

### 4.2. Ядро — без `use`

**Функции** (когда нет объекта):
```
print, sleep, range
read_file, write_file, list_dir, create_dir
copy_file, copy_dir, is_dir
parse_json, to_json
type_of
```

**Методы** (у объекта через точку):
```
// List
list.len(), list.filter(fn), list.map(fn), list.join(str)
list.sum(), list.first(), list.last(), list.sort(), list.reverse()

// String
s.len(), s.trim(), s.split(str), s.contains(str)
s.to_int(), s.to_float(), s.to_upper(), s.to_lower()
s.starts_with(str), s.ends_with(str), s.replace(old, new)

// Number
n.to_string(), n.abs()
```

**Алгоритм выбора (для AI):**

```
1. Смотришь ПЕРВЫЙ АРГУМЕНТ функции
2. Если тип СОСТАВНОЙ (List[T], struct, String, Channel) → МЕТОД
   Пример: data.map(...), s.len(), ch.recv()
3. Если тип СКАЛЯР (Int, Float, Bool) → ФУНКЦИЯ или ОПЕРАТОР
   Пример: add(1, 2), abs(x), x + y
4. Если функция I/O (print, assert, spawn) → всегда ФУНКЦИЯ
   Пример: print("..."), assert(true)
```

**Коротко:** Объект есть → `obj.method()`, нет объекта → `function()`.

### 4.3. Стандартная библиотека — `use std.xxx`

```

```

**В разработке:** Channel с таймаутами (`recv_timeout`).

## 5. Мутация — `mut`

### 5.1. Правило

`mut` в определении функции или метода — единственный маркер мутации. При вызове — всегда `.`. Компилятор генерирует `&` или `&mut` автоматически.

### 5.2. Функции

```
// чтение — без mut
fn total(list) -> Int
    return list.sum()
end

// мутация — mut в параметре
fn sort(mut list)
    list.sort_in_place()
end

// вызов — всегда .
print(total(data))
sort(data)
```

### 5.3. Методы

```
impl Counter
    fn get(self) -> Int
        return self.value
    end

    fn increment(mut self)
        self.value = self.value + 1
    end
end

c.get()
c.increment()
```

### 5.4. Генерируемый Rust

| U-код | Rust |
|-------|------|
| `fn show(user)` | `fn show(user: &User)` |
| `fn sort(mut list)` | `fn sort(list: &mut Vec<i64>)` |
| `show(user)` | `show(&user)` |
| `sort(data)` | `sort(&mut data)` |
| `fn get(self)` | `fn get(&self)` |
| `fn increment(mut self)` | `fn increment(&mut self)` |

### 5.5. Субъект действия

Метод (точка) — объект действует на свои данные: `text.trim()`.
Функция — внешнее действие: `print(text)`, `sleep(100)`.

### 5.6. I/O и чистота

- Функции могут выполнять I/O (чтение/запись файлов, сеть, `print`)
- `mut` маркирует только мутацию аргументов (изменение входных данных)
- Глобальные переменные отсутствуют — нет скрытого состояния между вызовами
- Побочные эффекты видны в коде явно: `write_file()`, `spawn()`, `print()`

## 6. Конкурентность

### 6.1. Spawn

```
spawn handle(conn)

fn worker(ch)
    loop
        task = ch.recv()
        process(task)
    end
end
spawn worker(ch)
```

### 6.2. Каналы

```
use std.channel: Channel

ch = Channel.new()
ch.send("задача")

// Бесконечное ожидание — не использовать без таймаута
msg = ch.recv()

// Предсказуемое ожидание с таймаутом — рекомендуется
msg = ch.recv_timeout(1000)  // 1000 мс = 1 сек
match msg
    Some(m) => process(m)
    None => print("timeout")
end
```

### 6.3. Правила spawn

- Внешние переменные видны на чтение
- Мутация внешних данных в spawn запрещена (компилятор проверяет по `mut`)
- Spawn ловит паники автоматически

```
data = [1, 2, 3]
spawn fn()
    print(data.len())     // ok — чтение
end

spawn fn()
    sort(data)            // ошибка: sort(mut list) — мутация внешнего
end
```

### 6.4. Автопараллельность for (экспериментально, исследуется)

Компилятор анализирует тело `for`: нет `mut` на внешних данных → итерации независимы → может параллелить автоматически. Решение по практике. На данный момент не гарантируется автоматическое распараллеливание — использовать явный `spawn` для гарантированной параллельности.

### 6.5. Реализация

`spawn` → `tokio::spawn` (~400 байт на задачу). Каналы → `tokio::sync::mpsc`.

## 7. Синтаксис

### 7.1. Как в Rust

```
fn, struct, trait, impl, enum, match + =>,
for, while, loop, if, else, elif, //, ->
```

### 7.2. Отличия от Rust

| U | Rust | Причина |
|---|------|---------|
| `end` | `}` | Надёжнее для ИИ-генерации |
| нет `;` | `;` | Меньше токенов |
| `fn(x) expr` | `\|x\| expr` | Клавиатура |
| `$(name)` | `format!("{name}")` | Меньше токенов |
| `mut` в сигнатуре | `&mut` при вызове | Проще — маркер только в определении |
| нет `let` | `let`/`let mut` | Меньше токенов |
| `and`, `or`, `not` | `&&`, `\|\|`, `!` | Читаемость |
| `[T]` дженерики | `<T>` | Клавиатура |
| `spawn f()` | `tokio::spawn(...)` | Одно слово |
| `return` явный | последнее выражение | Один способ |
| скрипт без main | `fn main()` | Меньше boilerplate |

```

**Pattern matching:**

```
// С enum вариантами
match maybe_number
    Some(n) => print("Число: $(n)")  // Some — шаблон
    None => print("Нет числа")      // None — шаблон
end

// Со списками
match list
    [] => 0
    [x] => x
    [x, ..rest] => x + sum(rest)  // x — первый, rest — остальные
end
```

**Важно:** `none` (строчное) — значение, `None` (заглавное) — шаблон в `match`:
```u
value = none                          // присваивание
match value
    None => print("пусто")           // сопоставление
end
```

**Обработка ошибок (без `?`):**

```

// ДА: явный match

    Ok(content) => content
    Err(e) => return Err(e)
end

// Или с дефолтным значением
content = match read_file("path")
    Ok(c) => c
    Err(_) => "default"
end
```

```
fn sum(list) -> Int

### 7.6. Модули

**Файловая система:**
```
project/
├── main.u
├── math.u          # модуль "math"
└── utils/
    └── http.u      # модуль "utils/http"
```

**Импорт:**

**Правила:**
- Имя модуля = путь без `.u`, `/` → `/`
- Корневой файл = `main.u`
- Всё публично внутри файла
- Циклические импорты — ошибка компиляции

**Циклы с диапазоном:**

```
// range(end) — от 0 до end-1
for i in range(5)      // 0, 1, 2, 3, 4

// range(start, end) — от start до end-1
for i in range(1, 6)   // 1, 2, 3, 4, 5
```

## 8. Интеграция с Rust

`.rs` файлы — полноценные модули в проекте:

```rust
// fast_math.rs
pub fn fibonacci(n: u64) -> u64 {
    if n <= 1 { return n; }
    let (mut a, mut b) = (0u64, 1u64);
    for _ in 2..=n { let tmp = a + b; a = b; b = tmp; }
    b
}
```

```
// main.u
use fast_math: fibonacci
print("fib(50) = $(fibonacci(50))")
```

U для 90% кода. Rust для 10% — парсеры, драйверы, SIMD, криптография.

## 9. Стратегия памяти

### 9.1. Принцип

Программист не думает о памяти. Компилятор выбирает оптимальный механизм. `mut` даёт информацию.

### 9.2. Что работает сейчас

- Stack-first — структуры на стеке
- Borrowing — нет `mut` → `&`, есть `mut` → `&mut`
- Move — последнее использование → move

### 9.3. Исследовано, не реализовано

| Механизм | Источник |
|----------|---------|
| Escape-анализ | Go, Java JIT |
| Reuse / FBIP | Perceus (Koka) |
| Compile-time RC elision | Lobster |
| Arena | Verona |
| COW | Swift |
| Auto-weak | Новое |

Реализация — когда появится конкретная задача.

### 9.4. Циклические структуры

Базовый язык не поддерживает циклические ссылки. Компилятор выдаёт ошибку при обнаружении цикла.

**Пример ошибки:**
```
error: Cyclic reference detected
  --> main.u:1:1
   |
 1 | struct Parent
 2 |     child: Child
   |
 5 | struct Child
 6 |     parent: Parent
   |     ^^^^^^^^^^^^^^ creates cycle Parent -> Child -> Parent
   |
   = help: Use std.linked for doubly-linked lists or store ID instead of reference
```

**Решения для сложных случаев (DOM, текстовые редакторы):**
- Использовать модули `.rs` (Rust) с raw pointers
- Использовать `std.linked` — реализовано на `unsafe` внутри, но API безопасно
- Использовать `std.arena` для временных графов

**Будущее:** Если язык взлетит и потребуется нативная поддержка циклов, она будет добавлена. Сейчас — используйте Rust для сложных структур данных.

### 9.5. Реализация в транспиляторе

Транспилятор U-lang реализует автоматическое управление памятью:

- **Compile-time cycle detection** — анализ AST перед генерацией кода
- **Zero-cost для нециклических структур** — move semantics, стек по умолчанию
- **RC для shared ownership** — где нужно shared, генерируется `Rc<RefCell<T>>`

**Ключевые слова:** 30 (без `weak`)
- Удалено: `weak` — не нужно для базового языка

**Статус:** Реализовано и протестировано — циклы обнаруживаются на этапе компиляции

## 10. Решаемые проблемы

### 10.1. Проблемы Rust

| Проблема | Решение U |
|----------|-----------|
| Сложный синтаксис (lifetimes, `&mut`) | `mut` в сигнатуре, компилятор генерирует |
| Медленная компиляция | Кэш, умные зависимости |
| Крутая кривая обучения | Скрипт без main/use/типов |
| Boilerplate | Нет `let`, `;`, `fn main()` |

### 10.2. Проблемы Python

| Проблема | Решение U |
|----------|-----------|
| Медленный (4.5x) | Нативная компиляция через Rust |
| GIL — нет параллельности | spawn, реальные потоки |
| None → крэш в рантайме | Option[T], ? при компиляции |
| pip hell, деплой | Один бинарник |

### 10.3. Проблемы Go

| Проблема | Решение U |
|----------|-----------|
| GC паузы | Нет глобального GC |
| if err != nil | явный match для Result |
| Нет enum/ADT | enum + match => |
| Нет доступа к Rust-экосистеме | .rs файлы, crates.io |

## 11. Чего не будет

- Интерпретатор. Только компиляция.
- Наследование. Трейты и композиция.
- Null/nil. `Option[T]` с `?`.
- Исключения. `Result[T, E]` с `?`.
- Множественный синтаксис. Один способ.
- Глобальный GC.
- Виртуальная машина / байткод.
- Extension functions. Методы через `impl`.
- Перегрузка функций. Уникальные имена.
- Implicit conversions. Явные преобразования.
- Макросы. Явный код.
- Глобальное mutable состояние.
