# Спецификация единого языка программирования

## Рабочее название: U (Universal)

Черновик v1.0 — март 2026

---

## 1. Предпосылки

Все необходимые концепции программирования существуют. Задача — соединить их в единый инструмент. U — это Kotlin для Rust: удобный язык поверх Rust-экосистемы. Не замена Rust, а партнёр.

### 1.1. Ключевое преимущество

Один проект — от скрипта до bare metal. U для 90% кода (серверы, CLI, скрипты). Rust для 10% (парсеры, движки, драйверы). `.u` и `.rs` файлы в одном проекте, один build, бесшовные вызовы. Вся экосистема crates.io доступна.

### 1.2. Принцип

Синтаксис Rust — за основу. Отличия от Rust — только где обосновано клавиатурой или упрощением. Один способ написать каждую конструкцию.

## 2. Режимы

### 2.1. Обзор

| Режим | Память | Конкурентность | Для чего |
|-------|--------|----------------|----------|
| Ничего не указал | RC+GC | spawn → tokio | Скрипты |
| `memory(auto)` | RC+GC | spawn → tokio | Серверы, CLI, проекты |
| `.rs` файл | Ownership | Полный Rust | Движки, парсеры, драйверы |

Скрипт и auto — синтаксис U. Критичный код — Rust как есть. Один проект, один бинарник.

### 2.2. Градиент

1. Скрипт — ничего не указал, всё доступно.
2. `memory(auto)` + `use` — проект, явные зависимости.
3. `.rs` — полный Rust для горячих модулей.

### 2.3. Совместимость

U вызывает Rust — бесшовно (U транспилируется в Rust). Rust вызывает U — тоже (U-функция = Rust-функция после транспиляции). Один Cargo-проект.

## 3. Мутация — оператор `::`

### 3.1. Принцип

`.` — чтение или команда внешней системе. `::` — изменение своих байтов в RAM.

### 3.2. Примеры

```
// чтение через .
users.len()
users.first()
db.exec("insert...")      // команда БД
conn.write(response)      // команда ОС
ch.send("task")           // команда каналу

// мутация через ::
list::push(4)
list::sort()
user::name = "Иван"
cache::set("key", value)
```

### 3.3. Функции и методы

```
fn sort(::list)
    list::sort_in_place()
end

sort(::users)             // :: при вызове

// методы
impl List
    fn len(self) -> Int
        self.items.count()
    end

    fn sort(::self)
        self.items::sort_raw()
    end
end
```

### 3.4. Spawn — запрет :: на внешних данных

```
data = [1, 2, 3]
spawn fn() print(data.len())     // ok: чтение
spawn fn() data::push(4)         // ошибка компиляции
```

Spawn ловит паники и ошибки автоматически.

## 5. Синтаксис

### 5.1. Как в Rust (без изменений)

```
fn, struct, trait, impl, enum, pub, match + =>,
for, while, loop, if, else, elif, //, ?, ->
```

### 5.2. Отличия от Rust — обоснованные

| U | Rust | Причина |
|---|------|---------|
| `end` | `}` | Клавиатура: `{}` требует латиницу |
| нет `;` | `;` | Упрощение: меньше шума |
| `fn(x) expr` | `\|x\| expr` | Клавиатура: `\|` недоступна |
| `$name` | `format!("{name}")` | Упрощение: короче |
| `::` мутация | `&mut` | Уникальная фича |
| нет `let` | `let`/`let mut` | Упрощение: не нужен |
| `and`, `or`, `not` | `&&`, `\|\|`, `!` | Клавиатура: `&&` `\|\|` недоступны |
| `!` unwrap | `.unwrap()` | Упрощение: короче |
| `[T]` дженерики | `<T>` | Клавиатура: `<>` перегружены с операторами сравнения |
| `.` для путей | `::` для путей | `::` занят мутацией |
| `spawn f()` | `tokio::spawn(async { f().await })` | Упрощение |
| `return` явный | последнее выражение | Читаемость: один способ |
| `memory(auto)` | ручное | Упрощение: RC+GC автоматически |
| скрипт по умолчанию | `fn main()` обязателен | Упрощение |

### 5.3. Блоки — `end`

```
if x > 0
    do_something()
elif x == 0
    do_other()
else
    handle()
end
```

### 5.4. Функции

```
fn add(a, b)
    return a + b
end

// pub — с типами и -> обязательно
pub fn process(data: List[Int]) -> Result[Int, Error]
    return data.filter(fn(x) x > 0).sum()
end

// процедура — нет return
fn log(message)
    file.write(message)
end
```

### 5.5. Лямбда — одно выражение

```
data.filter(fn(x) x > 0).map(fn(x) x * 2)
```

### 5.6. Enum и match

```
enum Shape
    Circle(radius: Float)
    Rect(width: Float, height: Float)
end

match shape
    Circle(r) => pi * r * r
    Rect(w, h) => w * h
end
```

### 5.7. Структуры, трейты, impl

```
struct User
    name: String
    age: Int
end

trait Display
    fn to_string(self) -> String
end

impl Display for User
    fn to_string(self) -> String
        return "$(self.name), $(self.age)"
    end
end
```

### 5.8. Строковая интерполяция

```
greeting = "Привет, $name! Возраст: $(user.age + 1)"
```

### 5.9. Импорты

```
use std.http: Server, Response
use std.db: Sqlite
```

Обязательны только с `memory(...)`. Без `memory` — стандартная библиотека доступна. Нет `*`. Алиас через `as`. Неиспользованный — ошибка.

### 5.10. Вызовы — скобки обязательны

```
handler = process       // ссылка
result = process()      // вызов
```

### 5.11. Логические операторы

```
if x > 0 and y < 10
    do_something()
end

if not found or expired
    refresh()
end
```

## 6. Конкурентность

### 6.1. U (скрипт и auto) — spawn + tokio

```
spawn handle_connection(conn)
spawn fn() send_email(user.email, "Привет")

ch = Channel.new()
spawn worker(ch)
ch.send("задача")
result = ch.recv()
```

### 6.2. Rust (.rs) — полный контроль

Thread, Mutex, async/await, tokio — всё что Rust умеет.

### 6.3. Гарантии

`::` запрещён в spawn на внешних данных. Spawn ловит паники автоматически. Горутина падает — остальные продолжают.

## 7. Система сборки

Cargo-модель. `u build`, `u run`, `u test`. `.u` транспилируется, `.rs` компилируется, линкуется вместе.

```
[build]
dev     = { backend = "cranelift", opt = 0, incremental = true }
release = { backend = "llvm", opt = 3, lto = true }
script  = { mode = "interpret" }
```

## 9. Происхождение

| Концепция | Источник |
|-----------|----------|
| Синтаксис, типы, `?`, `=>`, `->`, `enum`, `pub` | Rust |
| `struct`, `trait`, `impl`, `match`, `for/while/loop` | Rust |
| `::` мутация, spawn safety | Новое |
| RC+GC автоматический | Python (CPython), Swift (ARC) |
| `end` блоки | Ruby, Lua |
| `$` интерполяция | Kotlin, Shell |
| `and/or/not` | Python, SQL |
| Горутины + каналы | Go (модель), tokio (реализация) |
| Скрипт по умолчанию | Python, Kotlin (.kts) |
| U + Rust в одном проекте | Kotlin + Java (модель) |

## 10. Чего не будет

- Наследование. Трейты и композиция.
- Null/nil. `Option[T]` с `T?`.
- Исключения. `Result[T, E]` с `?`.
- Множественный синтаксис. Один способ.
- Расширяемые аннотации. 4 модификатора.
- Мутабельные переменные. Мутация через `::`.
- Многострочные лямбды. Одно выражение.
- Свой ownership синтаксис. Для этого Rust (.rs).

---

## 11. Рекомендации компилятору

Арена автоматически. Инлайнинг. Hot/cold. Escape-анализ. Дедупликация мономорфизации. RC по умолчанию, GC для циклов. Copy-on-write. Spawn → tokio.

## 12. Открытые вопросы

1. Границы U↔Rust — автоматические преобразования типов.
2. Стандартная библиотека — состав.
3. `!` force-unwrap — предупреждение компилятора?
4. Конкурентный доступ к структуре App из горутин.
5. Isolate (BEAM-подобный). Возможно не нужен — `::` + catch достаточно.
6. Persistent data structures / MVCC для данных в RAM.
7. Mesh-горутины — прямое общение без центрального канала.
8. Маппинг ошибок rustc → .u файл.
