# Спецификация единого языка программирования

## Рабочее название: U (Universal)

Черновик v0.8 — март 2026

---

## 1. Предпосылки

Современное программирование разделено на языки не по техническим, а преимущественно по социально-экономическим причинам. Конкуренция корпораций порождает дублирование, привязка к платформам создаёт искусственные барьеры, стихийность разработки ведёт к расточительности труда.

Все необходимые концепции уже существуют и проверены. Задача — соединить их в единый инструмент.

### 1.1. Основное противоречие

Контроль над ресурсами и удобство разработки — противоположности. Решение: переключение режимов в рамках одного языка.

### 1.2. Принцип проектирования

Один синтаксис. Два режима памяти. Один способ написать каждую конструкцию. Градиент от скрипта к проекту.

---

## 2. Базовый фундамент

За основу — Rust. Ownership, система типов, LLVM, Cargo. Реализация через транспилятор: наш синтаксис → Rust-код → rustc компилирует.

Что меняется по сравнению с Rust: два режима памяти, упрощённый синтаксис, горутины, `::` для мутаций, скриптовый режим по умолчанию.

---

## 3. Режимы работы файла

### 3.1. Скрипт (по умолчанию)

Файл без `memory(...)` в начале — скрипт. `memory(auto)` автоматически. Стандартная библиотека доступна без `use`. Код выполняется сверху вниз, без `main`. Имя модуля — из имени файла.

```
// hello.u — просто работает
name = "Андрей"
print("Привет, $name!")
```

```
// report.u — скрипт с функциями
data = parse_csv(read_file("workers.csv")?)

by_dept = data
    .group_by(fn(row) row.string("department"))
    .map_values(fn(rows) rows.avg(fn(r) r.float("salary")))

for (dept, avg) in by_dept
    print("$dept: $(avg.round(2))")
end
```

### 3.2. Проект (явный режим)

Файл с `memory(...)` в первой строке — проект. Импорты через `use` обязательны. Загружается только то, что указано — быстрее компиляция, меньше бинарник, прозрачные зависимости.

```
// server.u — проект
memory(auto)

use std.db: Sqlite
use std.http: Http, Router, Request, Response
use std.io: print

// ... код с явными зависимостями
```

```
// driver.u — проект с полным контролем
memory(own)

use hardware.gpio: Pin
use std.io: print

// ... код с ownership
```

### 3.3. Правило

Нет `memory(...)` → скрипт, всё автоматически. Есть `memory(...)` → проект, `use` обязателен. Один файл, одно расширение `.u`.

## 4. Управление памятью — два режима

### 4.1. Режимы

| Режим | Что делает компилятор |
|-------|----------------------|
| `memory(auto)` | RC для большинства объектов, GC для циклов. Программист не думает о памяти. |
| `memory(own)` | Ownership. Проверяет владение и времена жизни. Максимальная производительность. |

### 4.2. Градиент

Код из `memory(own)` всегда работает в `memory(auto)`. Обратное — не всегда, но компилятор объяснит что исправить. Начинаешь с `auto`, профилировщик показал проблему — переключаешь горячий модуль на `own`.

### 4.3. Управление предсказуемостью в `memory(auto)`

Без `weak` → RC + GC для циклов → просто, редкие микропаузы.
С `weak` где циклы → чистый RC → предсказуемо, без пауз.

## 5. Мутация — оператор `::`

### 5.1. Принцип

Иммутабельность по умолчанию. `.` — чтение. `::` — мутация. Каждая строка явно говорит что происходит.

### 5.2. Методы

```
users = [3, 1, 2]

count = users.len()        // . чтение
first = users.first()      // . чтение
users::sort()              // :: мутация
users::push(4)             // :: мутация
```

### 5.3. Поля

Чтение через `.`, запись через `::`:

```
name = user.name                    // . чтение
health = state.player.health        // . чтение
state.player::health = health - 10  // :: запись
state.player::name = "Новое имя"    // :: запись
```

### 5.4. Цепочки

```
// чтение
result = users
    .filter(fn(x) x.age > 18)
    .sorted()
    .first()

// мутация
users
    ::sort()
    ::reverse()
    ::truncate(10)
```

### 5.5. Функции

`::` перед аргументом — в определении и при вызове:

```
fn sort(::list)
    list::sort_in_place()
end

sort(::users)                 // :: при вызове — данные изменятся
process(users)                // без :: — данные не изменятся
```

### 5.6. Методы — определение

`::self` — метод мутирующий:

```
struct List
    fn len(self): Int                  // чтение
        return self.items.count()
    end

    fn sort(::self)                    // мутация
        self.items::sort_raw()
    end

    fn sorted(self): List              // чтение, возвращает новый
        copy = self.copy()
        copy::sort()
        return copy
    end
end
```

### 5.7. Переприсвоение локальных переменных

Свободное. Локальная переменная никому не видна, переприсвоение безопасно:

```
x = 5
x = 10     // ok
```

## 6. Модификаторы

Фиксированный список. Расширить нельзя.

| Модификатор | Значение | Применяется к |
|-------------|----------|---------------|
| `unsafe` | Обход проверок безопасности | Только к строке |
| `weak` | Слабая ссылка, предсказуемое освобождение | К значению |
| `test` | Тестовая функция | К функции |
| `public` | Видимость за пределами модуля | К функции |

`unsafe` — только на строку. Много unsafe подряд — вынеси в функцию с безопасной обёрткой.

## 7. Синтаксис

### 7.1. Блоки

Ключевое слово `end`. Форматировщик восстанавливает отступы по `end`.

### 7.2. Спецсимволы

**Допустимые** (без переключения раскладки): `?`, `!`, `:`, `=`, `>`, `<`, `.`, `,`, `()`, `[]`, `"`, `$`, `::`.

**Запрещённые** (требуют латиницы): `{}`, `<>`, `|`, `&`, `#`, `@`, `~`, `->`, `=>`.

Логические операторы — словами: `and`, `or`, `not`.

### 7.3. Применение символов

- `?` после вызова — пробрасывание ошибки: `result = parse(data)?`
- `?` после типа — nullable: `User?`
- `!` — force-unwrap: `value = maybe_value!`
- `::` — мутация: `users::sort()`, `sort(::users)`, `user::name = "Иван"`
- `.` — чтение: `users.len()`, `user.name`
- `:` — типы, ветки match
- `$` — интерполяция: `"Привет, $name!"`, `"Сумма: $(a + b)"`
- `()` — вызовы (обязательны), группировка
- `[]` — дженерики, индексы, литералы списков

### 7.4. Вызовы

Скобки `()` обязательны. Без скобок — ссылка на функцию:

```
handler = process       // ссылка
result = process()      // вызов
```

### 7.5. Функции и лямбды

Именованная — стандартная форма:

```
fn add(a, b)
    return a + b
end
```

Лямбда — одно выражение, для цепочек:

```
data.filter(fn(x) x > 0).map(fn(x) x * 2).sum()
```

Лямбда — одно выражение. Нужно больше — именованная функция.

### 7.6. Return

`return` = возвращает значение. Нет `return` = процедура:

```
fn add(a, b)
    return a + b
end

fn log(message)
    file.write("$(timestamp()): $message")
end
```

### 7.7. Типизация

Компилятор выводит типы. `public` — с типами (обязательно), без `public` — без типов (обязательно):

```
fn helper(data)
    return data.filter(fn(x) x > 0).sum()
end

public fn process(data: List[Int]): Result[Int, Error]
    return helper(data)
end
```

### 7.8. Импорты

Один способ — `use`. Обязателен только в проектном режиме (с `memory(...)`):

```
use std.db: Sqlite, Database
use std.http: Http, Router, Request, Response
use std.json: parse as parse_json
```

Нет `*`. Всегда явное перечисление. Неиспользованный импорт — ошибка компиляции. Алиас через `as`.

### 7.9. Базовые конструкции

```
// условия
if x > 0
    do_something()
elif x == 0
    do_other()
else
    handle_negative()
end

// циклы
for item in collection
    process(item)
end

while condition
    step()
end

// алгебраические типы
type Result[T, E]
    Ok(T)
    Err(E)
end

type Shape
    Circle(radius: Float)
    Rectangle(width: Float, height: Float)
end

// pattern matching
match shape
    Circle(r): pi * r * r
    Rectangle(w, h): w * h
end

// структуры
struct User
    name: String
    age: Int
end

// трейты
trait Printable
    fn to_string(self): String
end

impl Printable for User
    fn to_string(self)
        return "$(self.name), $(self.age)"
    end
end

// строковая интерполяция
greeting = "Привет, $name! Тебе $(age + 1) в следующем году."

// nullable и ошибки
config = read_file("config.toml")?       // ? — пробросить ошибку
user = find_user(1)!                      // ! — крэш если none
name = user?.name                         // ? — nullable доступ
```

## 8. Конкурентность

### 8.1. memory(auto) — горутины

I/O блокирует горутину, не поток ОС. Рантайм переключает. Программист не думает об async.

```
fn main()
    server = Http.listen(":8080")?
    loop
        conn = server.accept()?
        spawn handle_connection(conn)
    end
end
```

`spawn` принимает вызов функции или лямбду (одно выражение):

```
spawn handle_connection(conn)                          // именованная
spawn fn() send_email(user.email, "Привет")            // лямбда
spawn fn() log("обработано: $(process(item))")          // лямбда
```

Нужно больше одного выражения — именованная функция.

Каналы:

```
ch = Channel.new()
spawn worker(ch)
ch::send("задача")
result = ch.recv()
```

Ошибки в горутинах: `?` работает. Ошибка → рантайм логирует, горутина завершается. Нужна особая обработка — оберни функцию:

```
fn safe_handle(conn)
    match handle_connection(conn)
        Ok(_):  pass
        Err(e): conn.write(Response.error(500, "$e"))
    end
end

spawn safe_handle(conn)
```

### 8.2. memory(own) — async/await и потоки

Нет рантайма. Полный контроль:

```
memory(own)

use std.async: async, await
use std.thread: Thread

async fn process_stream(input, output)
    loop
        chunk = await input.read(256)
        processed = apply_effects(chunk)
        await output.write(processed)
    end
end

fn parallel_compute(data)
    chunks = data.split(4)
    threads = chunks.map(fn(c) Thread.spawn(fn() compute(c)))
    results = threads.map(fn(t) t.join())
    return results.flatten()
end
```

### 8.3. Параллельные коллекции — в обоих режимах

```
results = data.parallel().map(fn(x) heavy_compute(x)).collect()
```

### 8.4. Гарантии безопасности

Система типов запрещает гонки данных. `::` помогает: компилятор видит каждую мутацию.

## 9. Система сборки

Cargo-модель: lockfile, семантическое версионирование, единый репозиторий.

```
[build]
dev     = { backend = "cranelift", opt = 0, incremental = true }
test    = { backend = "cranelift", opt = 1 }
release = { backend = "llvm", opt = 3, lto = true }
script  = { mode = "interpret" }
embed   = { backend = "llvm", no_std = true, opt = "size" }
```

Целевые платформы: нативный бинарник, WebAssembly, байткод для JIT.

## 10. Инструменты

- **REPL** — `memory(auto)`, стандартная библиотека доступна.
- **Отладчик** — с первого этапа. DAP. Пошаговая отладка, точки останова.
- **Профилировщик** — CPU, память, аллокации.
- **Форматировщик** — единый, как gofmt.
- **LSP** — автодополнение, навигация, рефакторинг.
- **Менеджер версий** — установка компилятора одной командой.

## 12. Чего не будет

- **Наследование классов**. Трейты и композиция.
- **Null/nil**. `Option[T]` с сахаром `T?`.
- **Исключения (try/catch)**. `Result[T, E]` с `?`.
- **Ленивость по умолчанию**. Явная через итераторы.
- **Неограниченные макросы**. С ограничениями.
- **Множественный синтаксис**. Один способ.
- **Расширяемые аннотации**. 4 модификатора.
- **Мутабельные переменные**. Мутация через `::`.
- **async в memory(auto)**. Горутины.
- **Корпоративная собственность**. Общественное достояние.
- **Многострочные лямбды**. Одно выражение. Больше — именованная функция.
- **`module` в начале файла**. Имя из имени файла.

## 16. Рекомендации разработчикам компилятора

- **Арена** — автоматически когда все аллокации в scope умирают вместе.
- **Инлайнинг** — компилятор решает.
- **Hot/cold** — горячие пути оптимизируются.
- **Escape-анализ** — объект не покидает функцию → на стек.
- **Дедупликация мономорфизации** — идентичные инстанциации объединять.
- **Стратегия памяти в auto** — RC по умолчанию, GC для циклов без weak, арена где возможно, стек где возможно.
- **Copy-on-write** — копировать только при мутации (`::` даёт точную информацию).
- **Горутины** — рантайм с планировщиком, переключение при I/O-блокировке.

## 18. Путь реализации

### 18.1. Фаза 1 — Транспилятор

Наш синтаксис → Rust-код → rustc. Парсер, генератор, обёртка над cargo, маппинг ошибок. Один разработчик, месяцы.

### 18.2. Фаза 2 — Реальные проекты

CLI, сервер, обработка данных. Проверка синтаксиса на практике. Доработка спецификации по фактам.

### 18.3. Фаза 3 — Собственный компилятор

Bootstrapping: компилятор на самом языке. Свой borrow checker, рантайм, LSP. Только если фаза 2 подтвердила.
