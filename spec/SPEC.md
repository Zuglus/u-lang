# Спецификация единого языка программирования

## Рабочее название: U (Universal)

Черновик v1.2 — март 2026

---

## 1. Позиционирование

U — Kotlin для Rust. Удобный язык поверх Rust-экосистемы.

- U для 90% кода: серверы, CLI, скрипты, обработка данных
- Rust для 10%: парсеры, движки, драйверы, SIMD, криптография
- `.u` и `.rs` в одном проекте, один build, бесшовные вызовы
- Экосистема crates.io доступна

### 1.1. Принцип проектирования

Синтаксис Rust — за основу. Отличия — только где обосновано клавиатурой или упрощением. Один способ написать каждую конструкцию. Простота = скорость (меньше абстракций → быстрее код).

### 1.2. Принцип проектирования фич (от отрицания)

Каждая фича проверяется: какую конкретную фрустрацию программистов она убирает? Не «что хотят» (абстракция), а «чего НЕ хотят» (конкретная боль).

## 4. Стратегия памяти

### 4.1. Принцип

Программист не думает о памяти. Компилятор выбирает минимально необходимый механизм для каждого объекта. `::` даёт компилятору информацию, которой нет у Swift, Go, Python.

### 4.2. Иерархия механизмов (от дешёвого к дорогому)

| # | Механизм | Стоимость | Когда |
|---|----------|-----------|-------|
| 1 | Стек | Бесплатно | Escape-анализ: объект не покидает функцию |
| 2 | Move | Бесплатно | Один владелец, без разделения |
| 3 | Reuse (Perceus-стиль) | Бесплатно | `::` мутация + объект используется последний раз → обновление на месте |
| 4 | Borrowing | Бесплатно | Нет `::` → ссылка без RC. ~95% доступов |
| 5 | Arena | Почти бесплатно | Объекты в scope умирают вместе → одно освобождение |
| 6 | COW | Дёшево | `::` мутация разделяемых данных → копия только при изменении |
| 7 | RC | Дёшево | Разделяемые данные между scope → инкремент/декремент |
| 8 | Auto-weak | Дёшево | Обратная ссылка при `::` на рекурсивных типах → weak автоматически |

### 4.3. Роль `::` в оптимизации памяти

`::` — не только маркер для программиста. Это информация для цепочки оптимизаций:

- Нет `::` на объекте → borrowing, нет RC инкремента
- `::` на объекте + последнее использование → reuse на месте (Perceus)
- `::` на разделяемом объекте → COW (копия только при мутации)
- `::` на поле рекурсивного типа → компилятор проверяет цикл → auto-weak
- Нет `::` между spawn и концом scope → ссылка без RC в горутину

Без `::` компилятор не знает где мутация и использует RC для всего (как Swift). С `::` — знает точно и оптимизирует.

### 4.4. Циклические ссылки — автоматический weak

Циклы возникают только через `::` мутацию ссылочных полей. Компилятор видит `::` на поле рекурсивного типа → анализирует:

| Код | Циклы возможны? | Стратегия |
|-----|----------------|-----------|
| Без `::` на ссылочных полях | Нет | Чистый RC |
| С `::`, тип не рекурсивный | Нет | Чистый RC |
| С `::`, тип рекурсивный | Да | Auto-weak на обратную ссылку |

Правило auto-weak: кто создан первым — strong. Обратная ссылка — weak автоматически. Предупреждение компилятора: «back-reference made weak automatically».

Если программист хочет взаимное владение — общий владелец:

```
group = Group(members: [a, b])    // group владеет обоими
a::friend = b                      // weak
b::friend = a                      // weak
// пока group жив — оба живы
```

### 4.5. GC

GC как запасной механизм — сохраняется в языке, но ожидается что auto-weak покроет большинство случаев. Если практика покажет ситуации где auto-weak недостаточен — GC включается для конкретных типов. Решение по данным, не заранее.

### 4.6. Влияние на горутины

`spawn` + `::` запрет на внешних данных → компилятор знает что горутина не мутирует внешнее → передаёт ссылку без RC инкремента. Для 35,000 req/sec — 35,000 RC операций сэкономлено в секунду.

### 4.7. Происхождение подходов

| Подход | Источник |
|--------|----------|
| Escape-анализ | Go, Java JIT |
| Reuse / FBIP | Perceus (Koka, Microsoft Research) |
| Compile-time RC elision | Lobster |
| Регионы / Arena | Verona (Microsoft Research), MLKit |
| COW | Swift, наш `::` |
| Auto-weak | Новое — основано на `::` информации |

## 5. Мутация — оператор `::`

### 5.1. Принцип

`.` — чтение или команда внешней системе. `::` — изменение своих байтов в RAM.

### 5.2. Примеры

```
// . чтение и команды
users.len()
db.exec("insert...")
ch.send("task")

// :: мутация в RAM
list::push(4)
list::sort()
user::name = "Иван"
```

### 5.3. Методы

```
impl List
    fn len(self) -> Int         // чтение
        return self.items.count()
    end

    fn sort(::self)             // мутация
        self.items::sort_raw()
    end
end
```

### 5.4. Функции — субъект действия

Метод (точка) — объект действует на свои данные: `text.trim()`, `list.filter(...)`.

Функция — внешнее действие над объектом: `print(text)`, `sleep(100)`, `read_file(path)`.

Не два способа сделать одно и то же. Два разных действия. Точка = impl где-то определён.

## 6. Конкурентность

### 6.1. Spawn и каналы

```
use std.channel: Channel

spawn handle(conn)
spawn fn() send_email(user.email, msg)

ch = Channel.new()
spawn worker(ch)
ch.send("задача")
result = ch.recv()
```

### 6.2. Гарантии

- `::` запрещён в spawn на внешних данных — компилятор проверяет
- Spawn ловит паники автоматически
- Разделяемое состояние не повреждено

### 6.3. Реализация

`u run` — `std::thread` + `std::sync::mpsc`.
`u build` — `tokio::spawn` + `tokio::sync::mpsc` (~400 байт на задачу).

### 6.4. Оптимизация spawn через `::`

| Без `::` | С `::` |
|----------|--------|
| Захват данных в spawn всегда RC инкремент | Ссылка без RC если нет мутации после spawn |
| Нет информации о мутации | Компилятор знает точно |

## 7. Синтаксис

### 7.1. Как в Rust

```
fn, struct, trait, impl, enum, pub, match + =>,
for, while, loop, if, else, elif, //, ?, ->
```

### 7.2. Отличия от Rust

| U | Rust | Причина |
|---|------|---------|
| `end` | `}` | Клавиатура |
| нет `;` | `;` | Упрощение |
| `fn(x) expr` | `\|x\| expr` | Клавиатура |
| `$name` | `format!("{name}")` | Упрощение |
| `::` мутация | `&mut` | Уникальная фича |
| нет `let` | `let`/`let mut` | Упрощение |
| `and`, `or`, `not` | `&&`, `\|\|`, `!` | Клавиатура |
| `!` unwrap | `.unwrap()` | Упрощение |
| `[T]` дженерики | `<T>` | Клавиатура |
| `.` для путей | `::` для путей | `::` занят мутацией |
| `spawn f()` | `tokio::spawn(...)` | Упрощение |
| `return` явный | последнее выражение | Один способ |
| скрипт по умолчанию | `fn main()` | Упрощение |

### 7.3. Блоки

```
if x > 0
    do_something()
elif x == 0
    do_other()
else
    handle()
end
```

### 7.4. Функции

```
fn add(a, b)
    return a + b
end

pub fn process(data: List[Int]) -> Result[Int, Error]
    return data.filter(fn(x) x > 0).sum()
end
```

### 7.5. Лямбда — одно выражение

```
data.filter(fn(x) x > 0).map(fn(x) x * 2)
```

### 7.6. Enum и match

```
enum Shape
    Circle(radius: Float)
    Rect(width: Float, height: Float)
end

match shape
    Circle(r) => pi * r * r
    Rect(w, h) => w * h
end
```

### 7.7. Структуры, трейты, impl

```
struct User
    name: String
    age: Int
end

trait Display
    fn to_string(self) -> String
end

impl Display for User
    fn to_string(self) -> String
        return "$(self.name), $(self.age)"
    end
end
```

### 7.8. Строковая интерполяция

```
greeting = "Привет, $name! Возраст: $(user.age + 1)"
```

### 7.9. Dot-методы встроенных типов

```
// String
s.len()  s.trim()  s.find("x")  s.replace("a", "b")
s.starts_with("#")  s.ends_with(".md")  s.contains("hello")
s.slice(0, 5)  s.slice_from(5)  s.split_lines()  s.int()

// List
list.len()  list.first()  list.last()  list.sum()  list.join(", ")
list.filter(fn(x) x > 0)  list.map(fn(x) x * 2)  list.is_empty()

// Int
n.abs()  n.to_string()
```

### 7.10. Импорты

```
use std.http: Server, Response
use std.db: Sqlite
```

Ядро — без `use`. Стандартная — `use std.xxx`. Внешнее — `use module: name`.

### 7.11. Логические операторы

```
if x > 0 and y < 10
    do_something()
end
```

## 8. Интеграция с Rust

`.rs` файлы — полноценные модули:

```rust
// fast_math.rs
pub fn fibonacci(n: u64) -> u64 {
    if n <= 1 { return n; }
    let (mut a, mut b) = (0u64, 1u64);
    for _ in 2..=n { let tmp = a + b; a = b; b = tmp; }
    b
}
```

```
// main.u
use fast_math: fibonacci
print("fib(50) = $(fibonacci(50))")
```

## 9. Модификаторы

| Модификатор | Значение |
|-------------|----------|
| `unsafe` | Обход проверок, только на строку |
| `weak` | Слабая ссылка (ручная, при необходимости) |
| `test` | Тестовая функция |
| `pub` | Видимость за пределами модуля |

`weak` сохраняется как ручной модификатор для случаев где программист хочет явного контроля. Auto-weak — решение компилятора, не программиста.

## 12. Чего не будет

- Наследование. Трейты и композиция.
- Null/nil. `Option[T]` с `T?`.
- Исключения. `Result[T, E]` с `?`.
- Множественный синтаксис. Один способ.
- Расширяемые аннотации. 4 модификатора.
- Многострочные лямбды.
- `memory(auto/own/no)`. Просто `.u` и `.rs`.
- Глобальный GC. Только targeted, если auto-weak недостаточен.
- Extension functions. Методы только через `impl`.
