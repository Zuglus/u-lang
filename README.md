# U Language

**Статус: исследовательский проект, приостановлен.**

Попытка спроектировать язык программирования для ИИ-ассистированной
разработки, с диалектическим методом в процессе проектирования.

## Что оказалось

В ходе работы выяснилось:

1. **Диалектика как метод проектирования** — продуктивна: поиск
   противоречий, движение от конкретного к абстрактному, единство
   противоположностей в дизайне.
2. **Диалектика как содержание грамматики** (`thing`, `process`,
   `being`, `processes`, `nothing`, `Becoming[T]`) — оказалась
   переименованием существующих концепций (`struct`, `fn`,
   конструктор, тело, `Drop`, `Option[T]`). Семантической новизны
   компилятору не добавила.
3. **Исходная боль** — «ИИ пишет код в инструментах которые я не
   понимаю» — оказалась разрешима без нового языка: достаточно
   ограничить ИИ конвенцией «минимальный код, stdlib, один файл,
   без фреймворков», и результат становится читаемым за 10-15
   минут. Проверено на задаче генератора статического сайта —
   один файл Python на ~130 строк.
4. **Второй путь к той же боли** — научиться читать существующие
   инструменты (Hugo-шаблоны, например). Это дешевле чем строить
   свой язык в тысячу раз и даёт владение результатом ИИ сразу.

## Почему приостановлен

Реальная боль, послужившая мотивацией, оказалась меньше
предлагаемого решения в несопоставимых масштабах. Строительство
нового языка — 5-10 лет работы. Разрешение исходной боли —
конвенция для ИИ + 20 минут обучения чтению Hugo.

Возобновление — когда появится конкретная задача которая
**действительно** требует нового языка, а не когда появляется
абстрактное желание «сделать своё».

## История

- `main` — текущее состояние: только эта заметка, `CLAUDE.md` и `spec/SPEC.md`
- `legacy` — полный код последней попытки реализации:
  транспилятор (pest parser, type_checker, generator, formatter),
  runtime (Sqlite, HTTP, Channels), примеры. ~10000 строк.
  Сохранено для возможного возврата.

  быть полезна как материал)
