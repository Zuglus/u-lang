# U-lang — Правила для ИИ

## Философия

**Предсказуемость важнее краткости.** Код должен быть понятен локально, без «угадывания» состояния.

---

## Переменные

### Правило: Все переменные immutable

```u
// ✅ Правильно
x = 10
print(x)        // гарантированно 10

// ❌ Неправильно  
x = 10
x = 20          // ОШИБКА: переменная уже объявлена
```

**Почему:** ИИ знает, что значение не изменится неожиданно. Локально понятно.

---

## Изменяемое состояние

### Паттерн: Обёртка (Wrapper)

Если нужно «изменяемое» значение — оборачиваем в структуру:

```u
// ✅ Правильно
struct Counter
    value: Int
end

c = Counter(value: 0)
c.value = 10        // Меняем поле, не переменную
```

**Почему:** Явно видно, что это изменяемое состояние. Можно добавить валидацию.

---

## Функции

### Правило: Всегда явные типы параметров

```u
// ✅ Правильно
fn add(a: Int, b: Int) -> Int
    return a + b
end

// ❌ Неправильно
fn add(a, b)        // Неоднозначно — типы Unknown
    return a + b
end
```

**Почему:** Контракт функции понятен локально. ИИ не «угадывает» типы.

---

## Структуры и поля

### Правило: Поля структур mutable

```u
struct Point
    x: Int
    y: Int
end

p = Point(x: 10, y: 20)
p.x = 30            // ✅ Меняем поле
// p всё ещё тот же объект!
```

**Разница:**
- Переменная `p` = имя/ссылка (immutable)
- Поле `p.x` = содержимое объекта (mutable)

---

## Параметры функций

### Правило: По умолчанию immutable, `mut` явно

```u
// Параметр неизменяем (по умолчанию)
fn scale(p: Point, factor: Int) -> Point
    // p.x = p.x * factor    // ❌ ОШИБКА
    return Point(x: p.x * factor, y: p.y * factor)
end

// Явное mut разрешает изменение
fn move_x(mut p: Point, dx: Int)
    p.x = p.x + dx          // ✅ Работает
end
```

**Почему:** Видно в сигнатуре — функция изменит аргумент или нет.

---

## Pattern Matching

### Правило: Явные варианты в match

```u
enum Shape
    Circle(radius: Int)
    Rect(width: Int, height: Int)
end

fn area(shape: Shape) -> Int
    match shape
        Circle(r) => return r * r * 3
        Rect(w, h) => return w * h
    end
end
```

**Важно:** Переменные `r`, `w`, `h` получают типы из определения enum.

## Типы

### Базовые

| Тип | Литерал | Примечание |
|-----|---------|------------|
| `Int` | `42`, `-10` | Целые числа |
| `Float` | `3.14` | Дробные числа |
| `String` | `"hello"` | Интерполяция: `"$var"` |
| `Bool` | `true`, `false` | Логический |
| `List[T]` | `[1, 2, 3]` | Список элементов типа T |

### Вывод типов

- **Локальные переменные** — выводятся из значения: `x = 42` → `Int`
- **Параметры функций** — **только явные**: `fn f(x: Int)`

---

## Конкурентность

### Правило: spawn + Channel

```u
fn worker(id: Int, ch: Channel)
    msg = ch.recv()
    print("Worker $id: $msg")
end

ch = Channel.new()
spawn worker(1, ch)     // Новый поток
ch.send("task")
```

**Безопасность:** Компилятор проверяет мутацию внешних данных в `spawn`.

## Чего избегать

| ❌ Не делать | ✅ Вместо этого |
|-------------|----------------|
| `fn f(a, b)` | `fn f(a: Int, b: Int)` |
| `x = 10; x = 20` | `struct Wrapper { value: Int }` |
| Скрытая мутабельность | Явное `mut` в параметрах |
| Неявные преобразования | Явные конструкторы типов |

---

## Итог для ИИ

**U-lang предсказуем:**
1. Переменная = константа в пределах блока
2. Хочешь изменить состояние → обёртка (struct)
3. Функция = явный контракт (типы параметров)
4. Мутация = видна в сигнатуре (`mut`)

Код понятен локально, без «моделирования состояния».
